# files — 홈랩 공용 파일 스토어 구현 계획

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Rust(axum/tokio)로 외장 SSD(`bulk-ssd` PVC)에 파일을 보관하는 홈랩 공용 blob 스토어를 만들고, 내부 API(서비스 업로드)·공개 다운로드 표면을 앱 리스너로 분리하며, OpenAPI 계약과 타입드 Node SDK로 소비할 수 있게 한다.

**Architecture:** 단일 Rust 프로세스가 **두 개의 TCP 리스너**를 띄운다 — 내부(`:8080`, `/api/*` 전체, API 키) / 공개(`:8081`, public 버킷 다운로드+카탈로그만, 인증 없음). 저장은 **content-addressed** — 바이트는 `/.objects/<sha256>`에 불변 저장하고, 키의 메타(`.meta.json`)가 sha를 가리키는 **단일 atomic 커밋 포인터**다(DB 없음). 배포는 홈랩 플랫폼 매니페스트(`homelab/platform/files/prod/`)로 `bulk-ssd` PVC를 `/data`에 마운트.

**Tech Stack:** Rust 2021, axum 0.8, tokio 1, tower-http, serde/serde_json, sha2, hex, subtle(상수시간), nix(statvfs), tokio-util(ReaderStream), thiserror, tracing. Node SDK: TypeScript + tsdown + vitest, GitHub Packages. 배포: kustomize + Gateway API HTTPRoute + bats.

**참조 설계:** `docs/plans/2026-06-30-files-design.md` (SSOT). A.5 적대적 리뷰 반영 포함. **단, 본 계획의 content-addressed 커밋 protocol은 설계 §4의 "데이터 파일=단일 커밋 마커" 기술을 대체·구체화한다**(Phase C 패스1 발견 반영 — 아래 "## Phase C dispositions").

### Task M1.1: bucket·key 검증

**Files:** Create `src/path.rs`

**Step 1 실패 테스트:** simple 허용; `["../x","a/../../etc","/abs","a//b","a/./b","."]` 키 거부; `foo.meta.json`·`.tmp` 거부; 제어문자(`\0`,`\n`) 거부; 버킷 슬래시 거부; `safe_object_path`가 루트 하위 유지.

**Step 3 구현:**

```rust
use std::path::{Path, PathBuf};
use crate::error::AppError;
const RESERVED_SUFFIXES: &[&str] = &[".meta.json", ".bucket.json"];
fn segment_ok(seg: &str) -> bool {
    !seg.is_empty() && seg != "." && seg != ".." && !seg.starts_with('.')
        && seg.chars().all(|c| c.is_ascii_alphanumeric() || matches!(c, '-' | '_' | '.'))
}
const RESERVED_BUCKETS: &[&str] = &["api", "healthz", "readyz"]; // (발견 P4-2) 공개 catch-all/헬스 경로 충돌 방지
pub fn valid_bucket(b: &str) -> Result<(), AppError> {
    if b.len() > 64 || !segment_ok(b) || RESERVED_BUCKETS.contains(&b) { return Err(AppError::BadRequest("invalid_bucket")); } Ok(())
}
pub fn valid_key(k: &str) -> Result<(), AppError> {
    if k.is_empty() || k.len() > 1024 || k.starts_with('/') { return Err(AppError::BadRequest("invalid_key")); }
    for s in RESERVED_SUFFIXES { if k.ends_with(s) { return Err(AppError::BadRequest("reserved_suffix")); } }
    if k.split('/').any(|s| !segment_ok(s)) { return Err(AppError::BadRequest("invalid_key")); } Ok(())
}
pub fn safe_object_path(root: &Path, bucket: &str, key: &str) -> Result<PathBuf, AppError> {
    valid_bucket(bucket)?; valid_key(key)?; Ok(root.join(bucket).join(key))
}
pub fn meta_path(object: &Path) -> PathBuf {
    let mut s = object.as_os_str().to_owned(); s.push(".meta.json"); PathBuf::from(s)
}
```

> `safe_object_path`는 **메타 파일 경로 계산**에만 쓴다(바이트는 content-addressed `.objects/`에 저장 — M3). segment_ok가 traversal을 차단하므로 존재하지 않는 경로라도 안전(canonicalize 불요). PASS → Commit `feat: 경로 안전 검증`.

```rust
use std::path::Path;
use tokio::io::AsyncWriteExt;

pub async fn write_atomic(target: &Path, bytes: &[u8]) -> std::io::Result<()> {
    let parent = target.parent().expect("target has parent");
    mkdir_p_durable(parent).await?;          // (발견 P3-1) 조상 디렉터리도 내구적으로 생성
    let tmp = parent.join(format!(".tmp-{}", unique_suffix()));
    {
        let mut f = tokio::fs::File::create(&tmp).await?;
        f.write_all(bytes).await?;
        f.sync_all().await?;                 // 파일 내용 durable
    }
    tokio::fs::rename(&tmp, target).await?;
    fsync_dir(parent).await?;                // rename(디렉터리 엔트리) durable
    Ok(())
}

/// 디렉터리 fsync — rename/삭제 후 디렉터리 엔트리를 크래시 내구화.
pub async fn fsync_dir(dir: &Path) -> std::io::Result<()> {
    let dir = dir.to_owned();
    tokio::task::spawn_blocking(move || std::fs::File::open(&dir)?.sync_all())
        .await.expect("join")
}

/// (발견 P3-1) 누락 디렉터리를 한 레벨씩 생성하고, 생성 직후 부모를 fsync해 새 엔트리를 내구화.
/// create_dir_all은 새 조상 디렉터리를 fsync하지 않아 새 버킷/중첩 키 첫 쓰기가 크래시에 유실될 수 있다.
pub async fn mkdir_p_durable(dir: &Path) -> std::io::Result<()> {
    let mut to_create: Vec<std::path::PathBuf> = Vec::new();
    let mut cur = Some(dir.to_owned());
    while let Some(p) = cur {
        if tokio::fs::try_exists(&p).await? { break; }
        cur = p.parent().map(|x| x.to_owned());
        to_create.push(p);
    }
    for d in to_create.iter().rev() {
        match tokio::fs::create_dir(d).await {
            Ok(()) => { if let Some(parent) = d.parent() { fsync_dir(parent).await?; } }
            Err(e) if e.kind() == std::io::ErrorKind::AlreadyExists => {}
            Err(e) => return Err(e),
        }
    }
    Ok(())
}

fn unique_suffix() -> String {
    use std::sync::atomic::{AtomicU64, Ordering};
    static N: AtomicU64 = AtomicU64::new(0);
    format!("{}-{}", std::process::id(), N.fetch_add(1, Ordering::Relaxed))
}
```

## M3 — content-addressed 저장소

> **Phase C 발견 1 반영(근본 해결).** 2파일(데이터+메타) 2회 rename은 원자적 동시 커밋이 불가해 "신 메타 + 구 데이터" 창이 생긴다. content-addressing으로 제거: 바이트는 sha로 이름붙인 **불변 blob**, 메타가 sha를 가리키는 **단일 atomic 커밋 포인터**. read는 메타 단일 파일만 보므로 신/구 중 하나만 관측 → desync 구조적으로 불가능. 같은 size 덮어쓰기도 다른 sha → 다른 blob. (그래서 read에 락 불필요.)

**저장 레이아웃:**

```
/data/
  .objects/<sha256>          # 불변 콘텐츠(content-addressed). 이름이 곧 무결성 검증자.
  <bucket>/
    .bucket.json             # { visibility, owner, createdAt }
    <key>.meta.json          # 커밋 포인터: { contentType, size, sha256, createdAt, uploadedBy }
```

**Step 3 구현:**

```rust
mod atomic; mod locks; pub mod reconcile;
pub use locks::KeyLocks;
use std::path::PathBuf;
use sha2::{Digest, Sha256};
use crate::error::AppError;
use crate::meta::{ObjectMeta, BucketMeta, Visibility};
use crate::path::{safe_object_path, meta_path, valid_bucket};

#[derive(Clone)]
pub struct Store { root: PathBuf, locks: KeyLocks }

impl Store {
    pub fn new(root: PathBuf) -> Self { Self { root, locks: KeyLocks::new() } }
    pub fn blob_path(&self, sha: &str) -> PathBuf { self.root.join(".objects").join(sha) }
    fn meta_for(&self, bucket: &str, key: &str) -> Result<PathBuf, AppError> {
        Ok(meta_path(&safe_object_path(&self.root, bucket, key)?))
    }

    pub async fn put(&self, bucket: &str, key: &str, content_type: &str, by: &str, bytes: Vec<u8>)
        -> Result<ObjectMeta, AppError>
    {
        let meta_path = self.meta_for(bucket, key)?;        // 검증 포함
        let sha = hex::encode(Sha256::digest(&bytes));
        let _g = self.locks.lock(&format!("{bucket}/{key}")).await; // 같은 키 쓰기 직렬화
        // 1) 불변 blob. 있으면 무결성 검증 후 재사용; 손상(sha 불일치)이면 덮어써 치유.
        //    (발견 P3-2: 무검증 dedup은 손상 blob을 재사용·서빙. 새 바이트를 이미 들고 있으므로 검증만 추가)
        let blob = self.blob_path(&sha);
        let intact = matches!(tokio::fs::read(&blob).await, Ok(b) if hex::encode(Sha256::digest(&b)) == sha);
        if !intact {
            atomic::write_atomic(&blob, &bytes).await.map_err(AppError::Internal)?;
        }
        // 2) 메타 = 단일 atomic 커밋 포인터
        let meta = ObjectMeta { content_type: content_type.into(), size: bytes.len() as u64,
            sha256: sha, created_at: crate::clock::now_rfc3339(), uploaded_by: by.into() };
        atomic::write_atomic(&meta_path, &serde_json::to_vec(&meta).unwrap()).await.map_err(AppError::Internal)?;
        Ok(meta)
    }

    pub async fn head(&self, bucket: &str, key: &str) -> Result<ObjectMeta, AppError> {
        let raw = tokio::fs::read(self.meta_for(bucket, key)?).await.map_err(|_| AppError::NotFound)?;
        let meta: ObjectMeta = serde_json::from_slice(&raw).map_err(|_| AppError::NotFound)?;
        if !tokio::fs::try_exists(self.blob_path(&meta.sha256)).await.map_err(AppError::Internal)? {
            return Err(AppError::NotFound); // 포인터가 가리키는 blob 부재 → 비서빙(reconciliation이 정리)
        }
        Ok(meta) // 락 불필요: 메타는 단일 atomic 파일, blob 불변
    }

    pub async fn get_bytes(&self, bucket: &str, key: &str) -> Result<(ObjectMeta, Vec<u8>), AppError> {
        let meta = self.head(bucket, key).await?;
        let bytes = tokio::fs::read(self.blob_path(&meta.sha256)).await.map_err(AppError::Internal)?;
        Ok((meta, bytes))
    }

    /// Range 스트리밍용 blob 핸들(메타와 함께).
    pub async fn open(&self, bucket: &str, key: &str) -> Result<(ObjectMeta, tokio::fs::File), AppError> {
        let meta = self.head(bucket, key).await?;
        let f = tokio::fs::File::open(self.blob_path(&meta.sha256)).await.map_err(|_| AppError::NotFound)?;
        Ok((meta, f))
    }

    pub async fn delete(&self, bucket: &str, key: &str) -> Result<(), AppError> {
        let mp = self.meta_for(bucket, key)?;
        let _g = self.locks.lock(&format!("{bucket}/{key}")).await;
        match tokio::fs::remove_file(&mp).await {     // 커밋 포인터만 제거 → 즉시 사라짐
            Ok(()) => { if let Some(p)=mp.parent(){ atomic::fsync_dir(p).await.map_err(AppError::Internal)?; } Ok(()) }
            Err(e) if e.kind()==std::io::ErrorKind::NotFound => Ok(()),
            Err(e) => Err(AppError::Internal(e)),
        }
        // blob은 reconciliation이 미참조 GC.
    }
}
```

### Task M3.5: reconciliation + blob GC

**Files:** Create `src/store/reconcile.rs`

`run_once(root, gc_grace)`:
1. 모든 버킷 서브트리를 **재귀** 순회 → `*.meta.json`에서 **참조 sha 집합** 수집(중첩 키 `bucket/a/b.zip.meta.json` 포함; `.bucket.json` 제외). **발견 1: 비재귀 글롭은 중첩 키 blob을 미참조로 오인해 삭제하므로 반드시 재귀.**
2. **2단계 tombstone GC**(발견 2·P4-3): 미참조 blob을 처음 관측하면 `.objects/.gc-pending.json`에 `sha→최초관측시각` 기록. 삭제는 **`now - 최초관측시각 > gc_grace`**인 것만(다시 참조되면 pending에서 제거). blob mtime이 아니라 **미참조 지속시간** 기준이라 (a) 갓 기록된 미커밋 blob, (b) 덮어써져 막 미참조가 된 구 blob(in-flight reader가 meta-read 후 blob-open 사이) **모두** grace 동안 보존된다.
3. `.tmp-*` 잔재도 **mtime이 `gc_grace`보다 오래된 것만** 삭제(활성 스트리밍 temp 보존).
4. **무결성**: blob `sha256(content)==파일명` 검증(부팅 1회 + 저속 백그라운드 스크럽), 불일치(bit rot) → `.objects/.corrupt/` 격리 + 경고.

`gc_grace` 기본 1h(config `FILES_GC_GRACE`); 최대 업로드 시간 ≪ grace 전제.

**테스트:** 중첩 키 `a/b.zip` blob이 GC에서 **생존**; 미참조+오래된 blob → GC; 미참조+**최근** blob → 보존; 이름≠해시 → 격리; 오래된 `.tmp-*` → 삭제, 최근 → 보존. Commit `feat: reconciliation(재귀 참조수집 + grace-period GC + 무결성)`.

use subtle::ConstantTimeEq; use sha2::{Digest, Sha256};
#[derive(Clone, serde::Deserialize)]
#[serde(rename_all = "camelCase")]   // (발견 P4-1) 문서 keys.json은 writeBuckets/readBuckets(camelCase)

    #[serde(default)] pub write_buckets: Vec<String>, #[serde(default)] pub read_buckets: Vec<String>, #[serde(default)] pub admin: bool }
#[derive(Clone)] pub struct KeyRegistry { keys: Vec<ApiKey> }
impl KeyRegistry {
    pub fn load(path: &std::path::Path) -> std::io::Result<Self> {
        let keys = serde_json::from_slice(&std::fs::read(path)?)
            .map_err(|e| std::io::Error::new(std::io::ErrorKind::InvalidData, e))?;
        Ok(Self { keys })
    }
    pub fn authenticate(&self, bearer: &str) -> Option<&ApiKey> {
        let want = hex::encode(Sha256::digest(bearer.as_bytes()));
        let mut found = None;
        for k in &self.keys { if bool::from(k.sha256.as_bytes().ct_eq(want.as_bytes())) { found = Some(k); } } // 전체 순회(타이밍)
        found
    }
}
impl ApiKey {
    pub fn can_write(&self, b: &str) -> bool { self.admin || self.write_buckets.iter().any(|x| x==b) }
    pub fn can_read(&self, b: &str) -> bool { self.admin || self.read_buckets.iter().any(|x| x==b) }
}

### Task M5.1: 예약 기반 용량 가드

**Files:** Create `src/capacity.rs`

```rust
use std::path::Path; use std::sync::{Arc, Mutex};
use nix::sys::statvfs::statvfs;
use crate::error::AppError;

pub fn free_bytes(path: &Path) -> std::io::Result<u64> {
    let s = statvfs(path).map_err(|e| std::io::Error::new(std::io::ErrorKind::Other, e))?;
    Ok(s.blocks_available() as u64 * s.fragment_size() as u64)
}

#[derive(Clone)]
pub struct Capacity { root: std::path::PathBuf, min_free: u64, inflight: Arc<Mutex<u64>> }
impl Capacity {
    pub fn new(root: std::path::PathBuf, min_free: u64) -> Self { Self { root, min_free, inflight: Arc::new(Mutex::new(0)) } }
    /// reserve_bytes(Content-Length 또는 설정 max)를 예약. 체크+증가를 Mutex로 원자화(TOCTOU 제거).
    pub fn reserve(&self, reserve_bytes: u64) -> Result<Reservation, AppError> {
        let free = free_bytes(&self.root).map_err(AppError::Internal)?;
        let mut g = self.inflight.lock().unwrap();
        if free.saturating_sub(*g).saturating_sub(reserve_bytes) < self.min_free {
            return Err(AppError::InsufficientStorage);  // 507
        }
        *g += reserve_bytes;
        Ok(Reservation { inflight: self.inflight.clone(), bytes: reserve_bytes })
    }
}
pub struct Reservation { inflight: Arc<Mutex<u64>>, bytes: u64 }
impl Drop for Reservation { fn drop(&mut self) { *self.inflight.lock().unwrap() -= self.bytes; } } // 완료/실패 시 해제
```

쓰기 핸들러: `let _res = cap.reserve(content_length.unwrap_or(cfg.max_file_bytes))?;` 후 스트리밍. 테스트: 단일 거부(min_free>free) + **동시 N 업로드 overcommit**(각자 통과하나 합산 초과 시 일부 507, ENOSPC 없음 — M13.3). Commit `feat: 예약 기반 용량 가드(동시 overcommit 방지)`.
