# files — 홈랩 공용 파일 스토어

홈랩 서비스들이 API로 파일을 올리고 일부를 공개 다운로드하는 경량 blob 스토어.
DB 없이 파일시스템 위에서 객체-메타 일관성을 보장하는 것이 이 컨텍스트의 존재 이유다.

## Language

### 저장 모델

**객체(Object)**:
`bucket/key`로 주소되는 저장 단위. 커밋 포인터와 그것이 참조하는 블롭이 모두
존재할 때에만 "존재"한다.
_Avoid_: 파일(모호 — 블롭/포인터 어느 쪽인지 불명)

**블롭(Blob)**:
내용의 sha256으로 이름 붙는 불변 바이트열. 여러 객체가 같은 블롭을 공유할 수 있다(dedup).
_Avoid_: 데이터 파일, 원본

**커밋 포인터(Commit pointer)**:
객체의 존재를 정의하는 키별 사이드카 메타. 이것의 원자적 생성이 업로드의 커밋점이다.
_Avoid_: 메타파일, 사이드카(포인터 역할이 드러나지 않음)

**버킷(Bucket)**:
객체의 최상위 이름공간이자 가시성(public/internal)의 단위.

**Layout**:
데이터 루트 아래 모든 온디스크 이름·경로 규칙의 단일 소유자. 이름을 만들 줄도
(경로 계산·검증) 읽을 줄도(분류·커밋 포인터 순회) 아는 유일한 곳이며, Layout을
거치지 않은 경로 저작은 규칙 위반이다. 경계: **온디스크** 이름만 소유한다 —
표면의 라우트 모양은 Layout의 것이 아니다(그림자 라우트 참조).
_Avoid_: 경로 유틸, path helper, 스토리지 포맷

**예약 이름(Reserved names)**:
Layout이 자기 몫으로 점유해 사용자에게 불허하는 버킷명·키 접미사. **이름은 Layout의
것이지만, 그 이름이 공개 표면에서 갖는 모양은 아니다** — 표면 쪽 대응물은 그림자 라우트다.

**그림자 라우트(Shadow route)**:
예약 버킷명이 공개 표면에서 객체 경로로 오인되지 않도록 앞에서 가로채는 경로.
어떤 모양으로 가리느냐(그 이름 하나만이냐, 그 아래 전부냐)가 곧 관측 행동이므로
예약 이름 목록에서 기계적으로 파생되지 않는다 — 사람이 고르고, 두 목록의 정합성은
테스트가 강제한다.
_Avoid_: 404 라우트, 블록 라우트(무엇을 가리는지가 드러나지 않음)

### 일관성 & 수명

**non-servable**:
포인터와 블롭의 반쪽이 없거나 불일치해 서빙·목록에서 제외되는 객체 상태.
_Avoid_: 손상(격리 대상과 혼동)

**Reconciliation**:
저장소 불변식을 복원하는 주기 스캔 — temp 잔재 정리, tombstone GC, 비트로트 격리.
_Avoid_: 청소, 스캔(범위 불명)

**Tombstone GC**:
미참조 블롭을 ①최초 관측 시각으로 표시하고(tombstone) ②유예(grace)가 지나도록 무참조면
③**무덤을 거쳐** 회수하는 3단계 회수. 유예는 진행 중 업로드의 dedup 참조를 보호하고,
무덤 단계는 그 보호가 **닿지 못한** 진행 중 커밋을 정산(Settle)으로 건져낸다.
**회수는 무덤 단계에서 가역이다** — 정산이 회수를 확정하기 전에는 블롭 자체가 지워지지 않는다.

**격리(Quarantine)**:
내용 해시가 이름과 불일치하는 블롭(비트로트)을 삭제하지 않고 옆으로 치워 보존하는 처분.
_Avoid_: 삭제, 정리

### GC ↔ 업로드 경합

**핀(Pin)**:
업로드가 블롭을 **보기 전에** 잡는 무대기 예약. **커밋이 소유**하므로 취소가 떼어낼 수 없다.
**핀의 죽음 = 그 업로드의 종료 결과 확정**이다.
_Avoid_: 락, 레퍼런스 카운트(핀은 아무도 블록하지 않는다 — 업로드는 GC를 기다리지 않는다)

**착지(Landed)**:
**커밋 rename이 `Ok`를 반환했다** = 커밋 포인터가 실재한다. **GC의 유일한 보호 술어**이며
sticky하고 패스 스코프다.
_Avoid_: "커밋을 시도했다"(**착지가 아니다** — 시도까지 보호하면 실패한 업로드가 회수를 무한정 막는다),
"성공할 것 같다"(프록시는 결말이 아니다)

**무덤(Grave)**:
GC가 블롭을 지우기 **전에** 옮겨 두는 이름. 삭제를 **가역**으로 만드는 장치이며, 이름이 sha를
품으므로 복구가 자기완결적이고, 구 바이너리는 이 이름을 **모르는 것으로 취급해 건드리지 않는다**(롤백 안전).
_Avoid_: temp, 임시파일(**temp로 이름 붙이면 다음 패스가 그것을 청소해 버린다** — 무덤은 아무도 지우면 안 된다)

**코호트(Cohort)**:
**무덤을 만든 순간 살아 있던 핀들** — 고정·유한 집합. **대기 조건이지 보호가 아니다.**
무덤 이후에 생긴 핀은 들어오지 않는다(그 업로드는 블롭 부재를 보고 바이트를 재기록한다 → **자급자족**).
_Avoid_: 대기자 목록(누가 기다리느냐가 아니라 **무엇의 결말을 기다리느냐**가 요점)

**정산(Settle)**:
무덤에 대한 **유한·fail-CLOSED** 판정: **착지 확정 → 즉시 복원**(대기 0) / **코호트 드레인 → 착지 여부로
판정** / **예산 초과 → 복원 + 연기 + 에러 로그**. **보호 판정의 유일한 통로**이며 무덤을 **소비**한다.
결말을 **모르면 보존**한다.
_Avoid_: 확인, 체크(정산은 되묻지 않는다 — 결말을 **기다렸다가 관측**한다)

**재시작-필요 복구 계약(Restart-required recovery)**:
커밋이 핀과 키 락을 함께 소유하므로, 파일시스템 연산이 반환하지 않으면 **그 키는 프로세스가 재시작될
때까지 쓰기 불가**가 되는 계약. **의도된 교환이다 — 「잠김(가용성) < 되살아나기(무결성)」**: 가드를
타임아웃으로 놓으면 낡은 커밋이 **삭제된 키를 되살린다**(조용한 데이터 손상). 잠기더라도 **침묵하지는
않는다**(임계 초과 시 에러 로그).
_Avoid_: 데드락(교착이 아니다 — 상대를 기다리는 순환이 없다), 버그(선택한 교환이다)

### 접근

**표면(Surface)**:
내부(전체 API, 키 필수)와 공개(카탈로그·다운로드만)로 앱이 리스너 수준에서 강제
분리한 두 노출면.
_Avoid_: 엔드포인트 그룹, 라우트

**용량 가드(Capacity guard)**:
공유 쓰기 서비스로서 디스크 고갈 전에 업로드를 깔끔히 거부하는 free-space 방어선.
